Distributed Systems Development

In the second year of university in a group project, we built a web tool for developers. Users can login on the website with their github accounts, a list of all the users repositories are displayed that can be selected to automatiaclly convert and generate as a UML diagram. The generated diagram is editable by the user.

The goal with the project was creating a distributed system, we did this by following Blackboard architecture and using middleware to have a network of computers communicating between each other. More computers on the network increases processing power and to address fault tolerance if one of the servers crashes the system will automatically reconfigure the connections to keep running. Users that have access to same repositories and on the same network can preview all the changes/comments done to a diagram live without needing to refresh the page.

  • Date: Novemeber 2018
  • Technologies: HTML, CSS, Bootstrap, Javascript, Vue.js, Node.js, MongoDB, C
  • Github Repository: GitUML Repo
